Fundamentos de las bases de datos

El concepto o término de base de datos se conoció por primera vez en 1963 en un simposio celebrado en California, USA, en el cual se definió las bases de datos como una serie de información agrupada o estructurada; los modelos de datos se constituyen por una serie de conceptos que permiten describir las bases de datos, los cuales tienen como fin hacer entendible los datos almacenados.


Haga clic en cada ícono para conocer los conceptos que describen las bases de datos:

Una base de datos consiste en una gran cantidad de datos relacionados entre sí y a la vez almacenados en un soporte informático que tienen un significado implícito para el usuario final al estar interesados en conocer y utilizar su contenido con un propósito final determinado (Elmasri y Navathe, 2007).

Una base de datos es el conjunto de hechos representativos de situaciones, elementos, variables, informativos organizados en un mismo contexto para su uso y vinculación. Se le llama base de datos a los bancos de información que contienen datos relativos a diversas temáticas y categorizados de distinta manera, pero que comparten entre sí algún tipo de vínculo o relación que busca ordenarlos y clasificarlos en conjunto.

La principal función de una base de datos consiste en propiciar las condiciones para el almacenamiento y recuperación de la información necesaria, de manera que las diferentes personas dentro de una organización puedan hacer uso de ella especialmente tomando decisiones. Es por esta razón, que las bases de datos se convierten en un insumo esencial para la existencia de cualquier empresa, pues los datos se podrían llegar a concebir como el recurso más importante para las organizaciones, cómo si fueran el oxígeno para el funcionamiento del cuerpo humano. Elementos de las bases de datos:

En general, las bases de datos de uso cotidiano dentro de las organizaciones, se caracterizan por contar con al menos cuatro elementos: registro, campo, atributo y entidad.


Haga clic sobre cada botón (+) para conocer cada uno de los conceptos:

Registro: corresponde a cada fila que compone una tabla. Normalmente hace referencia al conjunto de datos que conforman un elemento diferenciado, asociado a cada uno de los atributos. En algunos casos pueden ser nulos en su almacenamiento, sobre todo cuando hay datos o campos faltantes.

Campo: es el dato que reposa en una celda, o dicho de otra forma, es el dato que se encuentra en la intersección (cruce) de una fila y una columna. Normalmente el contenido de un campo o celda por sí solo no dice nada, cobra sentido en la medida que se asocia a un contexto (registro y atributo).

Atributo: corresponde al nombre de la columna, que a su vez de manera conjunta le dan forma a la tabla. El atributo termina siendo la característica que identifica de manera exclusiva los registros contenidos en la base de datos. También se le conoce como Variable. Ejemplo: Nombre, Apellido, Teléfono.

Entidad: también es conocida como tupla, y hace referencia al conjunto de atributos o columnas agrupados en una misma temática: Ejemplo: estudiante (que a su vez contiene, nombre, apellido y teléfono).

Tipología de las bases de datos

Bases de datos según su flexibilidad:


Es una forma de concebir las bases de datos de acuerdo a su capacidad para ser modificadas de acuerdo a su función. En esta categoría están las bases de datos dinámicas y las bases de datos estáticas, que puede visualizar al dar clic en las siguientes tarjetas:

Bases de datos Estáticas:

Bases de datos Estáticas:

Estas son bases de datos de sólo lectura, normalmente utilizadas para almacenar datos históricos. Estas bases de datos no pueden modificar o actualizar los registros, campos, ni atributos, son usadas también para analizar hechos pasados o realizar proyecciones y tomar decisiones. Ejemplo: las bases de datos del archivo histórico.

Bases de datos Dinámicas:

Bases de datos Dinámicas:

Son las bases que están abiertas para ser actualizadas permanentemente, es decir, la información o los datos que contienen pueden ser modificados o actualizados en cualquier momento. Su modificación puede ser en tiempo real o no. Un ejemplo de una base de datos dinámica, puede ser la base de datos utilizada en un sistema de información de un supermercado, una tienda, un hospital o una empresa con operaciones permanentes, donde en tiempo real se están ingresando nuevos datos, modificando los existentes y eliminando los no utilizados.

Bases de datos según su modelo de gestión:


Es una forma de concebir las bases de datos de acuerdo a su estructura o forma como están organizadas. Las principales tipologías que se encuentran en esta categoría son: las bases jerárquicas, bases de datos en red, bases de datos relacionales, bases de datos deductivas, y las bases de datos multidimensionales. A continuación, se explican un poco cada una de ellas, haga clic sobre los botones para visualizar la información:


Bases de datos Jerárquicas:


Son aquellas que almacenan la información en una estructura jerárquica, en una forma similar a un árbol (visto al revés). A cada registro o ramificación del “árbol” en las bases de datos se les conoce como “nodo”. El nodo que no tiene padres es llamado raíz, y a los nodos que no tienen hijos se los conoce como hojas.

En el mundo corporativo, las bases de datos jerárquicas se usan principalmente cuando se administran grandes volúmenes de datos e información, así como cuando hay varios usuarios interviniendo a la vez desde procesos diferentes, permitiendo crear estructuras estables y de fácil gestión.

Bases de datos en red:


Este tipo de bases de datos, tiene bastante semejanza con las bases de datos jerárquicas, su diferencia radica en que permite que un mismo nodo tenga varios padres.

Estructuración de bases de datos

Para crear, diseñar y estructurar una base de datos es necesario comprender inicialmente las relaciones entre la información, el tipo de datos que serán incluidos, cómo se utilizarán y la forma en que tendrá que cambiar la organización para administrarlos. Requiere tanto de un diseño conceptual como físico. El diseño conceptual, también conocido como diseño lógico de la base de datos es un modelo abstracto y de ésta desde una perspectiva del negocio (propósito de la base de datos), mientras que el diseño físico muestra la verdadera disposición de la base partiendo de la propuesta para recopilar los datos, el modelo de almacenamiento, el sistema de procesamiento y la modalidad dispuesta para la visualización.


Para efectos de este curso, no distinguiremos entre diseño conceptual y diseño físico, mejor usaremos un término que integra los dos: Estructuración de Bases de Datos.


Estructurar los datos es el proceso en el cual se le da sentido, significado y valor a los datos para las diferentes situaciones y propósitos de la organización, en materia de gestión de la información, generación de conocimiento y toma de decisiones.

Gobernanza de los datos

En distintas partes de esta unidad, se ha recalcado y se seguirá insistiendo en que los datos son los activos corporativos o empresariales más importantes, incluso más que los edificios, la maquinaria y la propiedad intelectual. A pesar de ello, es un hecho que en la mayoría de las organizaciones no se les administra con el mismo rigor que los demás activos, en otras palabras, una gran mayoría de las organizaciones, al no darle el reconocimiento que se merecen los datos, no definen modelos administrativos para su explotación y gestión o no cuentan con una estrategia de gobernanza de los mismos.

Principios fundamentales en la Gobernanza

Cuando una organización diseña una estrategia o política para la gestión de las bases de datos (gobernanza) debe asegurarse cumplir con los siguientes principios, que puede visualizar al hacer clic sobre cada flecha:

Accesibilidad de los datos: Cuando se busca que el acceso a los datos será fácil y sencillo para los distintos usuarios, datos que, a su vez, serán oportunos y confiables.

Asegurar que los datos cumplen con las demandas: es decir, que tengan un uso y aplicación acertada para cualquier proceso empresarial.

Gestionar los datos como un activo: Facilita que los diferentes tomadores de decisiones de la organización entiendan el verdadero significado de los datos para el desarrollo y funcionamiento de la empresa. Recordemos que no todas las personas dentro de una organización son conscientes del verdadero significado y valor que tienen los datos para la gestión administrativa.

Asegurar la integridad: el gobierno de datos busca que haya coherencia, integridad y sobre todo articulación de los datos entre sí, de manera que se eviten ineficiencias y problemas en su gestión futura.

Generar oportunidades de negocio alternativas: estructurar nuevos esquemas de recopilación de datos, procesarlos y materializarlos como modelos de negocios alternativos.

Agregar valor: de hecho este es el principal propósito de la gobernanza de datos, puesto que facilita la definición de nuevas propuestas de valor para los diferentes usuarios (internos y externos) de la empresa. Todo esto facilita la gestión de la empresa y su transformación hacia niveles superiores.

Calidad de los datos

La calidad de los datos en una empresa u organización es un tema completamente indiscutible en la actualidad, y más aún cuando las condiciones del entorno digital y de interconexión se hacen cada día más presentes en los procesos administrativos. En este momento es impensable tomar decisiones empresariales, sin contar con esquemas estructurados para análisis de datos o generación de informes que faciliten abordar y comprender aspectos trascendentales de los distintos procesos al interior de la empresa y su entorno.


Es por esta razón, que, como administradores o gerentes de un proceso empresarial, debemos adoptar una filosofía que priorice la calidad de los datos como un elemento fundamental para el éxito administrativo, en donde se incluya lineamientos para la captura o registro, procesamiento, conservación y almacenamiento de los datos a analizar, todo esto enmarcado en la gobernanza de los datos.

Antes de comenzar a desarrollar el tema de calidad de los datos planeado para este curso, vamos primero a entender el concepto de calidad, clic en el ícono para conocer el concepto:


¿Qué es calidad?

Según lo que plantea la norma ISO 9000: 2000, la calidad se podría definir como “el grado en el que un conjunto de características inherentes cumple con los requisitos, esto es, con la necesidad o expectativa establecida, generalmente implícita u obligatoria".


Por otro lado, la calidad de datos podría entenderse como el esfuerzo donde se busque asegurarse que los datos utilizados y requeridos en los procesos de toma de decisiones de la empresa, cumplan con unos requisitos o estándares previamente definidos. Pueden ser requisitos de ubicación, descripción, digitación, almacenamiento, formato o incluso propiedades físicas del dato. La definición de los estándares de calidad, también es una tarea gerencial que debe abordarse con rigurosidad, puesto que allí se determinan condiciones como: disponibilidad, usabilidad, confiabilidad, pertinencia y legibilidad.

Volumen de un cubo


Como un cubo se caracteriza por tener sus lados, también llamados aristas, iguales y perpendiculares, para hallar el volumen se eleva al cubo la longitud de cualquiera de los lados (L) de la figura.


Paralelepípedo


Figura encerrada o delimitada por seis caras rectangulares que se interceptan perpendicularmente; sus caras son paralelas. Su volumen será la cantidad de unidades cúbicas que contiene; es el producto de sus tres lados o dimensiones.


Cilindro circular recto


Figura encerrada o delimitada por dos “tapas” circulares paralelas y una pared lateral de altura h, perpendicular a ellas.


Pirámide de base triangular


Figura encerrada o delimitada por cuatro caras triangulares. Su volumen será la cantidad de unidades cúbicas que contiene;

El volumen es la tercera parte del producto entre el área de la base y su altura.

Las características principales de las bases de datos jerárquicas son:


  • Diferentes usuarios pueden acceder a la información y compartirla sin muchos problemas.
  • Facilita la estructuración de los datos, y más aún cuando éstos son independientes.
  • Al ser una estructura rígida por su arquitectura, es muy difícil modificarla.
  • Requieren de sistemas de información con muy buena potencia y capacidad de procesamiento, especialmente, cuando tienen grandes volúmenes de datos.
  • Tienen muchas limitaciones por su incapacidad de representar de manera eficiente grandes cantidades de datos (redundancia de datos). Aunque con el tiempo, los avances tecnológicos han permitido que este problema sea mitigado en buena medida.
  • Entre más grandes sean, con ramificaciones y nodos distantes, se dificulta mucho más su acceso y procesamiento.

TEOREMA:


En una misma circunferencia o en circunferencias congruentes:


  1. Dos ángulos centrales son congruentes si subtienden cuerdas congruentes.
